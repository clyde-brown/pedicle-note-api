아래 제약조건을 토대로 문제정의서를 작성해주세요.

단, 다음 제약을 반드시 지키십시오.

[핵심 제약 조건]

1. 문제 정의의 중심은 반드시

   - “의사–환자 간 신뢰 붕괴”
   - “의사의 판단이 일방적으로 전달되는 구조”
   - “차팅이 의사에게 과도한 노동이며, 환자를 위한 문서 생성이 구조적으로 불가능한 상태”
     에 두십시오.
2. 문제를 ‘기능 부족’이나 ‘UX 불편’으로 축소하지 마십시오.

   - 클릭이 불편하다
   - 설명 화면이 없다
     같은 표현은 문제 정의에 사용하지 않습니다.
3. 문제 정의서는 반드시 다음 질문에 답해야 합니다.

   - 왜 지금의 의료 구조에서 신뢰가 깨질 수밖에 없는가?
   - 왜 의사의 판단은 환자에게 설명되지 못하는가?
   - 왜 의사는 환자를 위해 문서를 만들 수 없는가?
   - 이 문제가 반복될수록 의료 시스템은 어떤 방향으로 악화되는가?
4. 해결책(기능, 기술, AI, 구조)은 문제 정의서에 포함하지 마십시오.

   - “~할 수 있다”
   - “~를 제공한다”
     같은 표현은 금지합니다.
5. 문제 정의의 주체는 ‘의사’이며,
   환자는 결과를 겪는 존재로만 묘사합니다.
   (환자를 사용 주체로 설정하지 마십시오)

[산출물 형식]

다음 구조로만 작성하십시오.

1. 문제 배경 (현재 의료 환경에서 벌어지고 있는 구조적 상황)
2. 핵심 문제 정의 (한 문단, 하나의 문제로 수렴)
3. 문제가 발생하는 구조적 원인 (3~5개 불릿)
4. 이 문제가 방치될 경우의 결과 (의사 / 환자 / 의료 시스템 관점)
5. 왜 기존 시스템으로는 해결될 수 없는 문제인지
