# 페디클 프로젝트 문제 정의서 v1.0

> **타겟 명확화**: 페디클의 1차 타겟은 **중소 규모 개인병원(의원급 의료기관)**

## Problem Statement

### 문제 설명

현재 의료 기록 시스템은 **대형병원의 보험청구와 행정처리를 위해 설계**되어, 개인병원의 실제 진료 현실과 근본적으로 괴리되어 있다. **개인병원**에서는 한 명의 의사가 진료·차팅·환자 설명·행정 업무를 동시에 수행하면서도, **환자가 자신의 상태와 치료 내용을 충분히 이해할 수 있도록 설명**해야 한다는 이중 부담을 안고 있다. 그러나 기존 EMR은 이러한 **개인병원의 통합적 워크플로우와 환자 중심 소통**을 전혀 고려하지 않아, 오히려 진료 흐름을 방해하고 의사-환자 관계의 핵심인 **친밀한 소통**을 저해한다.

이로 인해 개인병원에서는 다음과 같은 악순환이 발생한다:

- 의사는 진료 중 차팅에 집중하느라 환자와의 아이컨택과 설명이 부족해짐
- 환자는 의사가 "컴퓨터만 본다"며 불만을 표출하고 충분한 설명을 받지 못함
- 재진 시 이전 판단 근거를 파악하지 못해 매번 처음부터 설명해야 함
- 복잡한 의학적 정보를 환자가 이해할 수 있는 형태로 변환하는 작업이 별도로 필요
- 결과적으로 **개인병원의 핵심 경쟁력인 '친밀하고 전문적인 진료'**가 실현되지 못함

### 발생 맥락

#### 개인병원 고유의 운영 특성

- **멀티태스킹 환경**: 한 명의 의사가 진료·차팅·환자 설명·처방·행정을 동시 수행
- **진료 속도와 품질의 딜레마**: 빠른 진료로 대기시간을 줄이면서도 충분한 설명과 친밀한 관계 유지 필요
- **환자와의 직접 관계**: 대형병원과 달리 의사-환자 간 **지속적이고 개인적인 관계**가 병원 경쟁력의 핵심
- **즉시 설명 요구**: 환자가 진료 과정에서 실시간으로 질문하고 설명을 요구하는 환경

#### 대형병원 중심 EMR의 구조적 부적합성

기존 EMR은 대형병원의 **분업화된 워크플로우**를 전제로 설계되어 개인병원 현실과 근본적 충돌:

**대형병원 워크플로우**: `진료 → (별도 시간) 차팅 → (별도 인력) 환자 설명 → (별도 부서) 행정처리`
**개인병원 워크플로우**: `진료 + 차팅 + 설명 + 처방을 동시 진행`

- 복잡한 메뉴 구조와 다단계 입력이 환자 대면 중 차팅을 방해
- 의학적 전문용어 중심 기록으로 환자 설명 시 별도 번역 작업 필요
- 청구 중심 항목들이 실제 환자 치료와 무관한 필수 입력 강요

#### 개인병원 정형외과 진료의 특수성

- **통증 맥락의 중요성**: 부위·유발조건·시간축·강도가 진단의 핵심이나 환자가 이해하기 어려운 의학용어로만 기록
- **재진 중심 진료**: 초진보다 재진이 많으며, **이전 진료 대비 변화 추적**이 치료 성패 결정
- **가설적 진단의 일상화**: 대부분의 근골격계 질환이 확정 진단까지 시간이 필요한 가설 상태로 진행
- **환자 참여 치료**: 운동·자세교정 등 환자의 **적극적 이해와 참여**가 치료 효과에 직결

#### 개인병원 환자의 기대와 현실

- **충분한 설명에 대한 기대**: 대형병원 대비 더 친밀하고 상세한 설명 기대
- **가족 설명 필요성**: 진료 후 가족이나 지인에게 설명할 수 있을 정도의 이해 요구
- **반복 문의**: 이해 부족으로 인한 전화 문의나 재방문으로 병원 운영 부담 가중

### 정량 지표 / Benchmark / 가설

#### 개인병원 특화 운영 지표

- **재진 준비 시간**: 개인병원에서 기존 기록 파악에 평균 3-5분 소요, 하루 50명 진료 시 **2.5시간 이상 손실** (업계 추정)
- **멀티태스킹 부담**: 개인병원 의사의 진료 시간 중 **차팅 40%, 환자 대면 60%**로 집중도 분산 (가설)
- **환자 대기시간**: 복잡한 EMR 사용으로 개인병원 평균 대기시간 **30분 증가** (대형병원 대비)

#### 의사-환자 소통 지표

- **환자 이해도**: 개인병원 진료 후 치료 내용 정확 이해률 **30% 미만**, 설명 요구로 인한 재방문/전화문의 증가 (의료 커뮤니케이션 연구 기반)
- **아이컨택 시간**: EMR 사용 중 의사-환자 아이컨택 시간이 전체 진료의 **40% 미만**으로 감소 (가설)
- **환자 만족도 격차**: 개인병원 환자의 **80%가 "충분한 설명"을 기대**하나 실제 만족도는 50% 미만 (가설)

#### 개인병원 EMR 부적합성 지표

- **시스템 부적합 체감도**: 대형병원 EMR 사용 시 개인병원 의사의 **70%가 "진료 흐름 방해"** 체감 (가설)
- **정보 손실률**: 이전 판단 근거의 **80% 이상이 추적 불가능** (자유텍스트 기록 한계)
- **데이터 재활용률**: 과거 유사 케이스 활용률 **5% 미만** (현행 자유텍스트 구조상 검색 불가)

#### 개인병원 경쟁력 손실 지표

- **환자 이탈률**: EMR 복잡성으로 인한 설명 부족 시 개인병원 **재진율 20% 감소** (가설)
- **의사 피로도**: 진료 후 추가 차팅 작업으로 **일평균 퇴근 1시간 지연** (개인병원 특성)

### 추론 근거

#### 업계 표준 연구 결과

- **EMR 기록 구조**: 의료 정보학 연구에 따르면 EMR 기록의 85% 이상이 비구조화 텍스트로 저장되어 분석 활용이 불가능 (JAMIA 2019)
- **의료 커뮤니케이션 격차**: 의사가 전달한다고 생각하는 정보와 환자가 실제 이해한 정보 간 **평균 40% 차이** (Patient Education Research)

#### 개인병원 vs 대형병원의 구조적 차이

**대형병원 환경**:

- 분업화: 진료→차팅→설명→행정이 각각 다른 시간/인력으로 처리
- 전문화: 각 과정에 특화된 인력과 시스템 지원
- 표준화: 정형화된 설명 자료와 프로토콜 존재

**개인병원 환경**:

- 통합화: 한 명의 의사가 모든 과정을 동시 처리
- 개인화: 환자별 맞춤 설명과 긴밀한 관계 유지 필요
- 유연성: 환자 질문에 즉시 대응 가능한 시스템 요구

**→ 기존 EMR은 대형병원 환경을 전제로 설계되어 개인병원에서는 구조적으로 부적합**

#### 개인병원 특화 논리적 추론 체계

**추론 1: 워크플로우 불일치 → 의사 부담 증가**

```
대형병원 EMR 구조
→ 순차적/분업적 입력 요구
→ 개인병원의 동시 다중작업과 충돌
→ 의사가 환자 대면 중 차팅 어려움
→ 진료 후 추가 차팅 시간 필요
→ 의사 피로도 증가 + 환자 대기시간 증가
```

**추론 2: 환자 소통 단절 → 개인병원 경쟁력 손실**

```
복잡한 EMR 인터페이스
→ 진료 중 화면 주시 시간 증가
→ 의사-환자 아이컨택 감소
→ 환자가 "의사가 컴퓨터만 본다" 불만
→ 개인병원 핵심 가치인 '친밀한 진료' 훼손
→ 환자 만족도 하락 → 재진율 감소
```

**추론 3: 의학정보 번역 부재 → 설명 효율성 저하**

```
의학 전문용어 중심 기록
→ 환자 설명 시 실시간 번역 필요
→ 설명 시간 연장 + 오해 가능성 증가
→ 환자 이해도 저하
→ 반복 문의 증가 → 병원 운영 부담
```

**추론 4: 임상 지식 축적 실패 → 장기 경쟁력 약화**

```
자유텍스트 기록 구조
→ 과거 진료 검색/분석 불가
→ 유사 케이스 경험 재활용 불가
→ 개별 진료의 반복
→ 의사 개인 역량에만 의존
→ 병원 차원의 임상 자산 축적 실패
```

### 문제 미해결 시 영향

#### 개인병원 즉시 영향 (1-3개월)

- **운영 효율성 저하**: 재진 준비 시간 증가로 일일 처리 환자 수 10-15% 감소
- **의사-환자 관계 악화**: "의사가 컴퓨터만 본다" 불만으로 환자 만족도 하락
- **차별화 실패**: 개인병원의 핵심 가치인 '친밀한 진료' 제공 불가
- **의사 스트레스 증가**: 진료 후 추가 차팅 작업으로 일일 1-2시간 업무 연장

#### 개인병원 중기 영향 (3-12개월)

- **환자 이탈 가속화**: 설명 부족과 소통 단절로 **재진율 20% 감소**
- **평판 하락**: 부정적 구전과 온라인 리뷰로 신규 환자 유입 차단
- **의료진 번아웃**: 지속적인 비효율적 업무로 의사 피로도 심화
- **대형병원 집중**: 환자들이 "설명 잘해주는 큰 병원"으로 이동하는 경향 강화

#### 개인병원 장기 영향 (1-3년)

- **경쟁력 영구 상실**: 대형병원 대비 개인병원의 고유 장점(친밀함, 접근성) 무력화
- **임상 자산 축적 실패**: 수년간의 진료 경험이 재활용 불가능한 텍스트로만 남음
- **시장 생태계 변화**: 개인병원의 역할 축소로 의료 접근성 저하
- **의료 혁신 저해**: 개인병원에서 생성되는 임상 데이터의 AI 학습 활용 불가

#### 의료계 전체 파급효과

- **의료 불평등 심화**: 대형병원 집중으로 지역별, 계층별 의료 접근성 격차 확대
- **1차 진료 체계 붕괴**: 개인병원의 게이트키퍼 역할 약화로 대형병원 과부하 가중
- **의료비 상승**: 개인병원에서 해결 가능한 문제의 대형병원 이동으로 의료비 상승

### Causality Analysis

**Symptom (현상)**

- 개인병원 의사가 진료/차팅/설명을 동시에 수행하며 시간 압박에 시달림
- 의사가 재진 시 이전 진료 내용을 파악하기 어려움
- 환자가 진료 후 치료 과정을 명확히 이해하지 못해 반복 문의
- 유사 환자 사례를 활용한 진료가 불가능

**Direct Cause (직접 원인)**

- EMR이 대형병원의 분업화된 청구/행정 업무 중심으로 설계됨
- 개인병원의 통합적 워크플로우를 고려하지 않은 시스템 구조
- 임상정보가 비구조화 텍스트로만 저장됨
- 진료 과정의 맥락과 판단 근거가 환자 설명 가능한 형태로 기록되지 않음

**Root Cause (3x Why 적용)**

**1차 Why: 왜 EMR이 개인병원에 부적합한가?**
→ EMR이 대형병원의 분업화된 워크플로우를 전제로 설계되어, 개인병원의 통합적 진료 환경과 충돌

**2차 Why: 왜 EMR 개발이 대형병원 중심인가?**
→ EMR 개발사들이 높은 구매력을 가진 대형병원을 주요 타겟으로 하여 수익 극대화를 추구하며, 개인병원은 시장 규모가 작다고 판단

**3차 Why: 왜 개인병원 시장이 외면받는가?**
→ 의료 정보화 정책이 "효율성과 표준화"를 중시하여 대형병원 모델을 기준으로 설계되었으며, **개인병원 고유의 가치인 "환자 중심 맞춤 진료"는 정책적 고려 대상에서 제외**

**근본 원인 (Root Cause)**:
**의료 정보화의 설계 철학이 "행정 효율성"을 우선하여, 개인병원의 핵심 경쟁력인 "의사-환자 간 긴밀한 소통과 맞춤형 설명"을 시스템 요구사항으로 인정하지 않는 구조적 맹점**

#### 근본 원인의 구체적 발현

- **정책 레벨**: 의료 질 평가가 정량적 지표(처리량, 표준화) 중심으로 설계되어 정성적 가치(환자 이해도, 만족도)는 부차적 취급
- **시장 레벨**: EMR 개발사의 사업 모델이 "많은 기능을 많은 사용자에게"로 설계되어, "특화된 기능을 특정 환경에"는 수익성이 낮다고 판단
- **기술 레벨**: 의료 정보 표준화가 대형병원의 분업 구조를 기준으로 설계되어, 개인병원의 통합적 워크플로우는 "비표준"으로 취급

## Persona Definition

### Persona 1: 개인병원 정형외과 의사

**Role**: 환자 진단, 치료 계획 수립, 직접 설명 및 차팅 수행
**Responsibility**: 정확한 진단, 효과적인 치료, 환자와의 긴밀한 소통, 신속한 진료 진행
**Decision Power**: 진단 확정, 치료 방법 선택, 추가 검사 지시, 진료 시간 배분

**개인병원 특수 상황**:

- 한 명의 의사가 진료/차팅/설명/처방을 모두 수행
- 대기 환자를 의식하며 진료 속도 유지 필요
- 환자와의 직접적이고 반복적인 소통이 병원 평판에 직결

**Failure Cost**:

- **Financial**: 진료 속도 저하로 일일 환자 수 감소 (하루 10명 감소 시 월 600만원 손실)
- **Operational**: 재진 시 이전 정보 재확인으로 진료 시간 연장 (진료당 3-5분 추가), 환자 설명 후 추가 차팅으로 퇴근 지연
- **Opportunity**: 축적된 임상 경험을 재활용하지 못해 진료 품질 개선 기회 상실
- **Compliance**: 의료법상 요구되는 진료 기록의 정확성 및 추적성 미충족 위험
- **Reputation**: 환자 설명 부족으로 인한 부정적 리뷰 및 환자 이탈

**Success Condition**:

- 환자 대면 중에도 자연스럽게 차팅 가능
- 재진 시 이전 판단과 근거를 즉시 파악 가능
- 환자 상태 변화를 시간축으로 명확히 추적
- 차팅 내용을 환자 설명 자료로 즉시 활용
- 유사 사례 경험을 현재 진료에 활용

### Persona 2: 개인병원 재진 환자

**Role**: 지속적인 치료를 통한 증상 개선 추구, 의사와의 긴밀한 관계 유지
**Responsibility**: 증상 변화 관찰 및 보고, 치료 지침 준수, 주변 지인에게 병원 추천
**Decision Power**: 치료 계속 여부 결정, 타 병원 방문 결정, 병원 평판 형성

**개인병원 환자 특성**:

- 대형병원보다 의사와의 **친밀하고 지속적인 관계** 기대
- 자신의 상태에 대한 **충분하고 이해하기 쉬운 설명** 요구
- 진료 후 가족이나 지인에게 설명할 수 있는 수준의 이해 필요
- 병원 선택 시 **의사의 설명 능력과 친절도**가 주요 기준

**Failure Cost**:

- **Financial**: 치료 과정 미이해로 인한 불필요한 재방문, 중복 검사, 타 병원 이동
- **Operational**: 매 진료마다 동일한 설명을 반복 요청, 전화 문의 증가로 병원 운영 방해
- **Opportunity**: 치료 과정 미이해로 적극적 참여 불가, 회복 속도 저하
- **Compliance**: 의료진 지시 사항을 정확히 이해하지 못해 치료 효과 감소
- **Trust**: 설명 부족으로 의사 신뢰도 하락, 부정적 구전으로 신규 환자 유입 저해

**Success Condition**:

- 진료 후 자신의 상태와 치료 계획을 **가족에게 설명 가능한 수준으로** 이해
- 이전 진료와의 차이점과 개선 사항을 명확히 파악
- 향후 치료 방향과 예상 경과에 대한 합리적 기대 형성
- 의사와의 소통에 만족하여 지속적인 재진 의향 및 타인 추천 의사

### Persona 3: 개인병원 원장 (경영자 겸 의사)

**Role**: 진료 수행과 동시에 병원 경영 관리
**Responsibility**: 직접 진료, 병원 수익성 관리, 환자 만족도 유지, 직원 관리, 경쟁 병원 대응
**Decision Power**: 의료정보시스템 도입, 진료 프로세스 개선, 투자 결정, 가격 정책

**개인병원 경영 특수성**:

- **의사이자 경영자**로서 진료 품질과 수익성을 동시에 고려
- 제한된 예산 내에서 최대 효율 추구
- 대형병원 및 주변 개인병원과의 직접 경쟁
- 환자 평판이 병원 생존에 직결

**Failure Cost**:

- **Financial**: 진료 효율 저하로 일일 환자 수 감소, 월 매출 10-20% 하락 위험
- **Operational**: 복잡한 EMR로 인한 직원 교육 비용 증가, 이직률 상승
- **Opportunity**: 축적된 임상 데이터를 활용한 진료 차별화 실패, 대형병원에 환자 빼앗김
- **Compliance**: 의료 질 평가 시 임상 성과 추적 및 개선 근거 부족
- **Competitive**: 환자 만족도 저하로 인한 평판 하락, 신규 환자 유입 감소

**Success Condition**:

- 의사 본인의 진료 효율성 향상으로 일일 처리 환자 수 20% 증가
- 환자 설명 품질 향상으로 재진율 및 추천율 상승
- 임상 데이터 자산화를 통한 **개인병원만의 전문성** 구축
- 대형병원 대비 **친밀하고 전문적인 진료**로 차별화
- 운영 효율화로 직원 만족도 상승 및 안정적 인력 유지

## Persona Conflict Report

### 🔴 핵심 충돌: 개인병원 의사 ↔ 시스템 데이터 품질 요구

**충돌 지점 설명**:
개인병원 의사의 **"환자와 눈 맞추며 진료하면서 동시에 차팅"** 필요와 시스템의 **"정확하고 완전한 구조화 데이터 입력"** 요구 간의 근본적 상충. 이는 개인병원에서만 발생하는 고유한 딜레마로, 대형병원처럼 진료와 차팅을 분리할 수 없는 환경적 제약이 원인

**개인병원 특수성**:

- 대형병원: 진료 → (시간 간격) → 차팅 분리 가능
- 개인병원: 진료 + 차팅이 동시 진행 필수 (환자가 바로 앞에 앉아있음)

**영향을 받는 지표**:

- 환자-의사 아이컨택 시간 vs 데이터 완성도
- 진료 중 환자 질문 대응 vs 정확한 실시간 입력
- 개인병원 핵심 가치(친밀감) vs 시스템 효율성

**단기 리스크**:

- 의사가 "환자보다 컴퓨터를 더 본다"는 환자 불만 급증
- 구조화 입력 부담으로 진료 시간 연장 → 대기 환자 불만
- 의사의 시스템 회피 → 자유텍스트 입력으로 회귀

**장기 리스크**:

- **개인병원 정체성 상실**: "친밀한 진료"라는 핵심 경쟁력 훼손
- 환자-의사 관계 질 하락으로 재진율 감소
- 시스템 사용 포기 → 투자비용 손실 + 문제 미해결

### 🔴 핵심 충돌: 환자 설명 요구 ↔ 개인병원 운영 효율성

**충돌 지점 설명**:
개인병원 환자가 기대하는 **"대형병원보다 더 친밀하고 충분한 설명"**과 개인병원의 **"제한된 인력으로 많은 환자를 처리해야 하는 경영 현실"** 간의 구조적 충돌. 이는 개인병원만의 독특한 딜레마로, 대형병원은 역할 분업으로 해결하지만 개인병원은 불가능

**개인병원의 이중 부담**:

- **환자 기대**: "개인병원은 큰 병원보다 더 친절하고 자세하게 설명해줄 것"
- **경영 현실**: "적은 수익으로 많은 환자를 봐야 생존"
- **의사 역할**: 진료·설명·차팅·처방을 혼자서 동시 수행

**대형병원과의 차이**:

- 대형병원: 간호사나 코디네이터가 별도 설명 담당
- 개인병원: 의사가 모든 설명을 직접 수행해야 함

**영향을 받는 지표**:

- 환자당 설명 시간 vs 일일 진료 환자 수
- 환자 만족도 vs 의사 피로도
- 병원 평판 vs 수익성

**단기 리스크**:

- 설명 시간 부족 → 환자 불만 → 부정적 평판 → 환자 감소
- 설명 시간 증가 → 대기시간 증가 → 대기 환자 불만 → 환자 이탈
- 의사 스트레스 증가로 진료 품질 편차 발생

**장기 리스크**:

- **개인병원 경쟁력의 역설**: 강점이어야 할 "친밀함"이 부담으로 작용
- 환자 이탈 → 수익 감소 → 더 많은 환자를 빨리 봐야 함 → 설명 시간 더 부족 (악순환)
- "설명 못해주는 개인병원 vs 체계적인 대형병원" 구도로 경쟁 열세

### Conflict Pair: 개인병원 원장 ↔ 시스템 도입 비용

**충돌 지점 설명**: 원장의 **효율적 시스템 도입 욕구**와 **제한된 투자 예산** 간 상충

**영향을 받는 지표**:

- 시스템 기능 완성도 vs 도입 비용
- 직원 교육 투자 vs 즉시 활용 가능성
- 장기적 경쟁력 vs 단기 수익성

**단기 리스크**:

- 저가 솔루션 선택으로 문제 미해결
- 고가 솔루션 도입으로 재정 압박
- 직원 교육 부족으로 시스템 활용도 저하

**장기 리스크**:

- 잘못된 시스템 선택으로 교체 비용 발생
- 경쟁 병원 대비 효율성 격차 확대
- 디지털 전환 실패로 시장 도태

## Metrics Definition

### Success Metrics

#### Primary North Star: 개인병원 핵심 가치 실현 지표

**🎯 "진료하며 차팅하며 설명하기" 동시 달성**

- **진료 중 실시간 차팅 완성률**: 환자 대면 중 차팅의 **85% 이상을 완료** (진료 후 추가 작업 최소화)
- **의사-환자 아이컨택 시간 비율**: 전체 진료 시간 중 **70% 이상 환자와 눈 맞춤** 유지 (차팅에 방해받지 않는 소통)
- **환자 설명 자료 즉시 생성률**: 차팅과 동시에 환자 이해용 시각 자료가 **자동 생성되는 비율 80% 이상**

**🎯 개인병원 운영 효율성 지표**

- **재진 준비 시간 단축률**: 이전 진료 파악 시간을 현재 3-5분에서 **30초 이내로 단축**
- **일일 진료 환자 수 증가**: 동일한 근무시간 내 **20% 이상 환자 수용 증가** (대기시간 단축 효과)
- **의사 퇴근 시간**: 진료 종료 후 **30분 이내 퇴근율 90%** (진료 후 추가 차팅 업무 최소화)

#### Secondary KPIs: 개인병원 경쟁력 강화 지표

**🎯 환자 관계 질 향상**

- **환자 치료 이해도**: 진료 후 "가족에게 설명 가능" 수준 **85% 이상** 달성
- **재진율**: 초진 환자의 재방문율 **70% 이상** 유지 (대형병원 대비 우위)
- **환자 추천율**: 기존 환자의 지인 추천으로 신규 환자 유입 **월 30% 이상**
- **전화 문의 감소율**: 진료 후 "이해 못했다" 류의 추가 문의 **60% 감소**

**🎯 개인병원 차별화 자산 구축**

- **유사 사례 활용률**: 과거 유사 케이스 참조 및 활용 빈도 **60% 이상**
- **임상 패턴 발견률**: 월별 새로운 임상 인사이트 발견 **5건 이상**
- **개인화 설명 자료 생성**: 환자별 맞춤 설명 자료 자동 생성 **90% 이상**

### Failure / Guardrail Metrics

#### 개인병원 Core Guardrails (절대 지켜야 할 경계선)

**🚫 의사-환자 관계 보호**

- **화면 주시 시간 제한**: 전체 진료 시간 중 **30% 이하**만 화면 주시 (환자와의 소통 우선)
- **진료 시간 증가 제한**: 구조화 입력으로 인한 진료 시간 증가 **3% 이내**로 제한
- **환자 불만 지표**: "의사가 컴퓨터만 본다" 불만 **월 5% 미만** 유지

**🚫 개인병원 운영 부담 방지**

- **학습 곡선 제한**: 신규 직원의 시스템 숙달 **3일 이내** (복잡한 교육 부담 차단)
- **추가 업무 방지**: 기존 EMR 대비 **추가 입력 항목 20% 이내**로 제한
- **의사 피로도**: 시스템 도입 후 의사 만족도 **80% 이상** 유지

#### Quality & Security Guardrails
