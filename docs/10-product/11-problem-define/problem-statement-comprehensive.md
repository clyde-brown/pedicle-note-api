# Problem Statement - 의료기록 구조화 플랫폼 (Pedicle Note API)

## Problem Statement

### 문제 설명
**신뢰 붕괴**가 의료 현장의 근본적 위기이다. 현재 의료기관의 차팅 시스템에서는 의사의 임상적 사고와 판단 근거가 구조화되지 않은 자유텍스트로만 저장되어, 환자가 진료 과정을 이해하기 어렵다. 이로 인해 '일방향 판단 → 결과 불만족 → 불신 누적'의 악순환이 반복되며, **누적되는 경험**이 의사-환자 간 신뢰를 지속적으로 약화시킨다. 이러한 신뢰 붕괴는 단순한 소통 문제가 아니라, 기존 EMR 시스템이 보험청구와 행정 목적에 최적화되어 환자 이해와 참여를 위한 투명성을 제공하지 못하는 구조적 한계에서 기인한다.

### 발생 맥락
- **대상 영역**: 정형외과 및 일반 임상 진료 현장
- **핵심 이해관계자**: 임상의사(진료), 환자(이해/신뢰), 의료기관(효율성/소송방지)
- **현 상황**: EMR 시스템이 행정·청구 목적으로 최적화되어 환자-의사 간 신뢰 구축을 위한 투명성 지원 부족
- **핵심 갭**: **차팅을 쉽게 하면서 동시에 환자용 설명 문서가 자동 생성되는 구조의 부재** - 의사는 차팅만 하고, 환자 설명은 별도로 해야 하는 이중 작업 구조
- **누적되는 경험**: 반복되는 일방향적 진료로 인해 환자의 불신이 누적되고, 이는 의료진의 업무 부담과 스트레스를 가중시키는 악순환 구조

### 정량 지표 / Benchmark / 가설
- **업계 표준**: 진료 기록 중 80% 이상이 자유텍스트 형태 (의료정보학 연구 참고)
- **추정 손실**: 진료당 평균 3-5분의 차팅 시간, 재진환자 이전기록 검토에 추가 2-3분 소요
- **가설**: 구조화된 차팅으로 전환 시 차팅 효율성 30% 개선 + 환자 이해도 50% 증가 가능
- **추론 근거**: 현재 의사들이 약어(HTN=고혈압) 사용하는 이유가 시간 절약 목적임을 감안

### 문제 미해결 시 영향
- **단기적**: 환자-의사 간 신뢰 관계 악화, 환자 만족도 저하, 의사 업무 부담 증가
- **중장기적**: 누적된 불신으로 인한 의료 분쟁 증가, 환자 이탈, 임상 지식 축적 실패로 인한 의료 품질 정체

### Causality Analysis

#### Symptom (현상)
1. 의사가 차팅을 "힘들고 귀찮은 작업"으로 인식
2. 환자가 진료 후 치료 내용을 충분히 이해하지 못함
3. 재진 시 이전 상태와의 변화를 파악하기 어려움
4. 임상 정보가 텍스트로만 저장되어 재활용 불가능

#### Direct Cause (직접 원인)
1. **차팅 시스템의 구조적 한계**
   - 자유텍스트 기반 입력 방식
   - 한정된 진료 시간 내 상세 기록의 어려움
   - 한글/영어 혼재로 인한 입력 복잡성

2. **의사결정 투명성 결여(근거 비가시화)로 인한 불신 누적 루프**
   - **일방향 판단 구조**: 의사의 임상적 사고와 판단 과정이 환자에게 가시화되지 않음
   - **근거 비가시화**: 치료 결정의 근거와 과정이 기록되지 않거나 환자가 접근할 수 없는 형태로만 저장
   - **설명 불가능성**: 결과가 부정적일 때 환자가 납득할 수 있는 형태의 설명 자료 부재
   - **차팅-설명 분리**: 차팅이 의사용 기록으로만 끝나고, 환자에게 전달 가능한 문서가 진료 중 생성되지 않음
   - **신뢰 누적 파괴**: 반복되는 불투명한 경험으로 인한 의료진 신뢰의 점진적 약화

3. **정보 구조화 부족으로 인한 소통 장벽**
   - 증상-판단-치료 간 연결고리 누락
   - 시간축 기반 변화 추적 시스템 부재
   - 가설적 진단과 확정 진단의 구분 부재

#### Root Cause (3x Why 적용)

**Why #1: 왜 신뢰가 누적적으로 무너지는가?**
→ 치료 근거와 과정이 환자에게 이해 가능한 형태로 남지 않아, 결과가 나쁠 때 '납득'이 불가능하기 때문

**Why #2: 왜 이해 가능한 형태로 기록이 남지 않는가?**
→ EMR 시스템의 'Source of Truth 대상'이 환자/임상사고가 아니라 **청구·법적 방어**로 설정되어, 환자 이해 가능한 '근거/맥락'은 설계 범위 밖으로 밀려났기 때문. 자유텍스트 자체가 문제가 아니라 **환자/임상사고를 위한 구조가 설계 범위 밖**이었던 것이 근본 원인

**Why #3: 왜 행정적 완결성이 소통보다 우선시되는가?**
→ **의료 시스템의 구조적 문제**: 의료기관의 수익 모델이 진료 품질이나 환자 만족도가 아닌 **보험청구 중심**으로 설계되어 있고, 규제 기관이 임상적 가치보다 법적 근거 마련을 우선하며, EMR 벤더들이 이러한 요구사항에 맞춰 행정 효율성을 우선한 시스템을 구축했기 때문

## Persona Definition

### Persona 1: 임상의사 (Primary User)
- **Role**: 환자 진료 및 의료기록 작성 담당
- **Primary Responsibility**:
  - 환자 상태 진단 및 치료 계획 수립
  - 법적 요구사항에 부합하는 진료기록 작성
  - 환자에게 치료과정 설명
- **Decision Power**:
  - 진료 방향 및 치료 방법 결정 (High)
  - 기록 시스템 도입/변경 요청 (Medium)
  - 환자 치료 우선순위 결정 (High)
- **Failure Cost**:
  - **Financial**: 진료 효율성 저하로 인한 환자 처리량 감소 (시간당 3-5명 → 2-3명)
  - **Operational**: 재진 환자 이력 파악 실패로 인한 진료 품질 저하
  - **Compliance**: 불충분한 기록으로 인한 의료분쟁 시 법적 리스크
  - **Opportunity**: 임상 경험 축적 및 사례 학습 기회 상실
- **Success Condition**:
  - 차팅 시간 30% 단축
  - 재진 환자 이전 상태 파악 시간 50% 단축
  - 환자 설명 만족도 증가

### Persona 2: 환자 (End Beneficiary)
- **Role**: 진료 서비스 수혜자 및 치료과정 참여자
- **Primary Responsibility**:
  - 정확한 증상 정보 제공
  - 처방된 치료 계획 준수
  - 치료 과정 이해 및 협조
- **Decision Power**:
  - 치료 방법 선택 동의/거부 (Medium)
  - 의료기관 선택 및 변경 (High)
  - 추가 검사/치료 수용 여부 (Medium)
- **Failure Cost**:
  - **Financial**: 부적절한 이해로 인한 치료 지연 및 추가 비용 발생
  - **Operational**: 치료 계획 불준수로 인한 회복 지연
  - **Opportunity**: 자신의 건강 상태에 대한 올바른 이해 기회 상실
  - **Compliance**: 의료진과의 소통 부족으로 인한 치료 결과 불만족
- **Success Condition**:
  - 진료 내용 이해도 80% 이상
  - 치료 과정 및 이유에 대한 명확한 파악
  - 의료진과의 신뢰관계 구축

### Persona 3: 의료기관 관리자 (Stakeholder)
- **Role**: 의료기관 운영 효율성 및 리스크 관리
- **Primary Responsibility**:
  - 진료 효율성 최적화
  - 의료 분쟁 예방 및 대응
  - 의료진 업무 환경 개선
- **Decision Power**:
  - 시스템 도입 예산 승인 (High)
  - 운영 프로세스 변경 권한 (High)
  - 의료진 교육/훈련 계획 수립 (Medium)
- **Failure Cost**:
  - **Financial**: 의료분쟁 소송비용, 차팅 비효율성으로 인한 수익 감소
  - **Operational**: 의료진 이직률 증가, 환자 만족도 저하
  - **Compliance**: 의료기록 부실로 인한 규제 기관 제재 위험
  - **Opportunity**: 임상 데이터 자산화 실패로 인한 경쟁력 상실
- **Success Condition**:
  - 의료분쟁 발생률 감소
  - 진료 처리량 증가
  - 의료진 업무 만족도 향상

## Persona Conflict Report

### Conflict Pair 1: 임상의사 vs 의료기관 관리자
- **충돌 지점**:
  - 의사는 진료 자율성과 기존 작업방식 유지를 선호
  - 관리자는 표준화와 효율성 최적화를 추구
- **영향을 받는 지표**: 시스템 도입 속도, 사용자 만족도
- **단기 리스크**: 시스템 도입 저항, 교육 비용 증가
- **장기 리스크**: 불완전한 도입으로 인한 기대효과 미달성

### Conflict Pair 2: 구조화 vs 진료 흐름
- **충돌 지점**:
  - 구조화된 입력은 정확성을 높이지만 입력 시간이 증가할 수 있음
  - 빠른 진료 흐름은 구조화를 제한할 수 있음
- **영향을 받는 지표**: 차팅 시간, 데이터 품질
- **단기 리스크**: 초기 학습비용으로 인한 진료 속도 저하
- **장기 리스크**: 부적절한 균형 설정 시 시스템 포기 가능성

## Metrics Definition

### Success Metrics

#### North Star Metrics (Trust Proxies)
- **신환 cohort 재방문율**: 신규 환자 cohort의 3개월 내 재방문율 75% 이상 → 85% 이상 (신뢰 회복 핵심 지표)
- **설명 후 'Teach-back' 성공률**: 환자가 자신의 언어로 치료 계획을 설명할 수 있는 성공률 80% 이상 (이해도 기반 신뢰 측정)
- **결정 공유 지표**: 치료 옵션 비교 화면을 보고 충분한 이해 후 동의까지 완료한 비율 85% 이상 (투명성 기반 신뢰 측정)
- **환자 설명서 생성 및 활용률**: 진료 후 5분 내 자동 생성 비율 90% 이상 + 24시간 내 환자 열람률 70% 이상 (차팅-설명 연동 핵심 지표)
- **불만 proxy 감소**: 추가 문의/민원/부정적 리뷰 발생률 월 5% 이하 (현재 15% 대비 감소) - 신뢰 붕괴 역지표

#### Key Performance Indicators
- **차팅 효율성**: 환자당 평균 차팅 시간 30% 단축 (현재 5-7분 → 목표 3-5분)
- **재진 효율성**: 재진 환자 이전 상태 파악 시간 50% 단축
- **치료 계획 참여율**: 환자의 치료 계획 적극적 참여 및 이행 비율 90% 이상 (운동/재활 체크인, 복약/예약 이행 등 행동 기반 측정)
- **데이터 재사용률**: 유사 증상 환자 사례 참조 활용 빈도 주당 10건 이상
- **ICD 코드 정확도**: 증상-부위 기반 ICD 코드 제안 정확도 85% 이상
- **시스템 만족도**: 의료진 시스템 사용 만족도 4.2/5.0 이상

### Failure / Guardrail Metrics

#### Critical Failure Indicators
- **진료 지연률**: 차팅으로 인한 진료 지연 발생률 5% 미만 유지
- **데이터 무결성**: 필수 임상 정보 누락률 2% 미만
- **시스템 가용성**: 진료시간 내 시스템 다운타임 0.1% 미만
- **규정 준수**: 의료기록 법적 요구사항 미준수 사례 0건

#### Early Warning Indicators
- **사용자 이탈**: 월간 활성 사용자 감소율 5% 초과 시 경고
- **입력 완성도**: 구조화 필드 완성률 70% 미만 시 UX 개선 필요
- **오류 발생률**: 사용자 신고 오류 주당 10건 초과 시 품질 점검
- **학습곡선**: 신규 사용자 온보딩 완료 시간 2주 초과 시 교육과정 재검토

#### Compliance Metrics
- **보안**: 환자 정보 무단 접근 시도 0건
- **감사**: 데이터 변경 이력 추적 가능률 100%
- **다중 테넌트**: 테넌트 간 데이터 격리 위반 0건
- **백업**: 일일 데이터 백업 성공률 99.9% 이상

## 완료 조건 검증

### ✅ 필수 통과 조건 확인

1. **측정 가능한 문제 정의**:
   - 차팅 시간, 구조화율, 이해도 등 모든 지표가 정량화됨
   - 업계 표준 및 추론 근거 제시

2. **3x Why 기반 Root Cause**:
   - 개인 문제 → 시스템 문제 → 구조적/환경적 문제로 심화 분석 완료
   - 보험청구 중심 의료시스템의 구조적 한계까지 도출

3. **페르소나별 실패 비용 2개 이상 범주**:
   - 모든 페르소나에 Financial, Operational, Compliance, Opportunity 비용 정의

4. **페르소나 간 이해관계 충돌 리포트**:
   - 의사 vs 관리자, 구조화 vs 진료흐름 충돌 명시

5. **지표 정합성**:
