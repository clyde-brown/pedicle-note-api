# 차트 페이지 기능 분석서

## 🎯 차트 페이지 기능 개요

차트 페이지는 의사의 **"3.진료시작 → 9.진료 종료"** 전체 프로세스가 진행되는 핵심 화면입니다. 페디클 시스템의 핵심 가치인 **S-T-D 구조화 차팅**과 **환자 설명 가능성**이 모두 실현되는 중심 페이지로서, 진료 흐름을 해치지 않으면서도 구조화된 임상 데이터를 생성하는 것이 목적입니다.

클릭 기반 선택은 UX 트릭이 아니라 **의사의 사고 흐름을 중단하지 않기 위한 설계 제약이다.**

### 차트 페이지의 핵심 원칙

1. **진료 흐름 최우선**: 어떤 기능도 의사의 진료 속도를 느리게 하지 않음
2. **환자 대면 모드**: 환자와 화면을 공유하며 설명 가능한 UI 제공
3. **점진적 구조화**: 최소 입력으로 시작하여 필요시 상세 정보 확장
4. **실시간 저장**: 입력 즉시 자동 저장으로 데이터 손실 방지

### 차팅 입력 방식 설계 원칙

**S-T-D 구조화 차팅의 핵심은 진료 흐름을 해치지 않는 입력 방식입니다.**

#### 1. 클릭 기반 선택 우선 원칙

- **S-T-D 각 축의 구성 요소들은 클릭 기반 선택을 기본 원칙**으로 함
- 타이핑보다 **선택(클릭)**이 더 빠르고 정확한 구조화 데이터를 생성
- 진료 중 입력 부담을 최소화하여 **진료 연속성 보장**

#### 2. 입력 방식 제약의 목적

- **진료 흐름 비침해**: 복잡한 입력 과정으로 진료가 중단되지 않도록 방지
- **데이터 정합성**: 선택 기반 입력으로 표준화된 구조화 데이터 확보
- **환자 대면 적합성**: 환자와 함께 보는 상황에서도 빠른 입력 가능

#### 3. UI 구현 비결정 원칙

- **이 원칙은 UI 구현 지시가 아님**을 명확히 함
- 구체적인 컴포넌트(버튼, 슬라이더, 드롭다운, 이미지 맵 등)는 UX/UI 설계 단계에서 결정
- PRD는 **"클릭 기반 선택"이라는 입력 방식의 제약**만 정의

### S-T-D 구조화 차팅 관련 기능

**S-T-D는 의사의 진료 사고 흐름을 구조화하는 3가지 핵심 축**이며, 각 축은 다음과 같은 구체적인 데이터 요소들로 구성됩니다.

#### S축 (Situation - 상황) 구성 요소

**기본 구성 요소 (F005 - Situation 입력)**

- **증상(Symptom)**: 정형외과 특화 50개 증상 마스터 기반 선택 (F051)
- **부위(Anatomy)**: 정형외과 해부학적 부위 50개 세분화 기반 선택 (F052)
- **발생 시점(Onset)**: 급성/아급성/만성 + 구체적 시작 시기
- **심각도(Severity)**: VAS, NRS 등 표준화된 통증 척도

**확장 구성 요소 (검사 결과 통합)**

- **신체검사 소견**: 이학적 검사 결과 구조화 입력 (F019)
- **관절 가동범위(ROM)**: 측정값 + 정상 범위 비교 (F020, F042)
- **근력 평가**: 0-5 등급 + 해부학적 매핑 (F021, F043)
- **특수검사**: 검사별 양성/음성 결과 + 시계열 추적 (F022, F044)
- **압통점**: 해부학적 위치 + 중증도 점수 (F023)
- **영상검사**: 구조화 판독 소견 + 비정상 소견 하이라이트 (F025, F047, F050)

#### T축 (Treatment - 치료) 구성 요소

**기본 구성 요소 (F006 - Treatment 입력)**

- **치료 계획**: 표준 치료 프로토콜 기반 선택
- **처방 정보**: 약물, 물리치료, 주사 등 치료 방식
- **가설 상태**: 확정되지 않은 치료 계획 기록 허용 (규약 6)

**확장 구성 요소**

- **치료 단계**: 1차/2차/3차 치료 순서
- **치료 목표**: 통증 완화/기능 회복/재발 방지 등
- **치료 강도**: 보존적/적극적/수술적 치료

#### D축 (Duration - 기간) 구성 요소

**기본 구성 요소 (F007 - Duration 입력)**

- **예상 회복 기간**: 일수/주수/월수 단위 예측
- **재진 일정**: 다음 내원 예약 시기
- **경과 예후**: 호전/유지/악화 예상

**확장 구성 요소**

- **시계열 비교**: 이전 내원 대비 변화 추이 (F035, F026)
- **마일스톤**: 치료 단계별 중요 시점 설정
- **위험 기간**: 악화 가능성이 높은 시기

#### S-T-D 간 연관성 매핑

- **S→T 연관성**: 증상-부위 조합에 따른 치료 프로토콜 자동 제안 (F055)
- **S→D 연관성**: 중증도와 부위에 따른 회복 기간 예측
- **T→D 연관성**: 치료 방식에 따른 경과 관찰 일정 조정
- **전체 통합**: S-T-D 패턴을 기반으로 한 유사 환자 조회 (F011)

## 🔍 차트 페이지 적합성 분석

### 차트 페이지에 **반드시 포함**되어야 하는 기능 (필수/핵심)

#### 1. 진료 세션 관리 기능

- **F004 (Encounter 생성)**: 차트 페이지의 시작점
- **F012 (진료 기록 저장)**: 차트 페이지의 종료점
- **F013 (진료 기록 수정)**: 사후 보완 기능

**적합성 근거**: 차트 페이지는 하나의 진료 세션(Encounter) 전체를 다루는 화면이므로 생성-저장-수정이 모두 필요

#### 2. Problem 중심 차팅 기능

- **F034 (Problem Unit 관리)**: 문제별 카드 구조
- **F035 (재진 변화 비교)**: 이전 진료와의 연속성

**적합성 근거**: 개별 문제(Problem) 단위로 진료를 구조화하는 페디클의 핵심 철학 구현

#### 3. S-T-D 구조화 입력 기능

**F005 (Situation 입력) - 상황 축 기록**

- **기본 정보**: 증상(F051 마스터) + 부위(F052 마스터) + onset + 심각도 *(클릭 기반 선택 우선)*
- **확장 정보**: 신체검사(F019), ROM(F020), 근력(F021), 특수검사(F022), 압통점(F023) 연계
- **영상검사**: 판독 소견(F025, F047) 및 시계열 비교(F026, F049) 통합
- **연관 제안**: 증상-부위 연관성 기반 관련 부위 자동 제안 (F055)

**F006 (Treatment 입력) - 치료 축 기록**

- **치료 계획**: 표준 프로토콜 기반 클릭 선택 + 가설 상태 허용 *(클릭 기반 선택 우선)*
- **처방 정보**: 약물/물리치료/주사 등 다중 치료 방식 조합 선택
- **치료 단계**: 1차/2차/3차 치료 순서 및 강도 설정
- **S축 연동**: 입력된 증상-부위에 따른 치료 옵션 자동 필터링

**F007 (Duration 입력) - 기간 축 기록**

- **회복 예측**: 부위별/중증도별 표준 회복 기간 선택 *(클릭 기반 선택 우선)*
- **재진 일정**: 치료 방식에 따른 추천 재진 간격 자동 계산
- **경과 예후**: 호전/유지/악화 예상 + 위험 기간 설정
- **T축 연동**: 선택된 치료에 따른 경과 관찰 포인트 자동 설정

**적합성 근거**: S-T-D는 페디클 시스템의 데이터 구조 핵심이며, 각 축의 세부 구성 요소들이 **서로 연관된 클릭 기반 선택**을 통해 일관성 있는 구조화 데이터를 생성해야 함. 각 축은 독립적이면서도 상호 연관성을 통해 진료 사고 흐름을 자연스럽게 구조화함.

#### 4. 환자 소통 기능

- **F031 (환자 대면 차팅 모드)**: 환자와 함께 보는 UI
- **F032 (환자 설명 자료 생성)**: 의학용어 자동 변환

**적합성 근거**: 페디클의 핵심 가치인 "환자 설명 가능성"을 실현하는 기능

#### 5. ICD 매핑 기능

- **F008 (ICD 자동 제안)**: S-T 기반 자동 코딩
- **F010 (ICD 코드 확정)**: 최종 진단 코드 매핑

**적합성 근거**: 구조화된 차팅의 최종 목적이 정확한 진단 코드 매핑이므로 차트 페이지에서 완성되어야 함

### 차트 페이지에 **조건부 포함**되는 기능 (보조)

#### 1. ICD 수동 검색 기능

- **F009 (ICD 수동 검색)**: 자동 제안 실패시 백업

**적합성 근거**: 기본적으로는 별도 검색 페이지에서 처리하나, 진료 흐름을 위해 모달 형태로 차트 페이지에 임베딩 가능

#### 2. 진료 요약 기능

- **F033 (진료 요약 PDF)**: 진료 완료 후 요약 생성

**적합성 근거**: 진료 종료 시점에 필요하므로 차트 페이지에서 바로 실행 가능해야 하지만, 별도 페이지로 분리도 가능

### 차트 페이지에서 **제외**되는 기능들

차트 페이지는 **하나의 Encounter(진료 세션)**에 집중하므로 다음 기능들은 별도 페이지에서 처리:

- **F002 (환자 검색)**: 환자 목록 페이지
- **F003 (환자 등록)**: 환자 등록 페이지
- **F011 (유사 환자 조회)**: 임상 도구 페이지
- **F014-F018 (관리자 기능)**: Admin Console
- **F019-F030 (검사 기능)**: 별도 검사 페이지들

## 🔄 기능 간 연관성 및 워크플로우

### 차트 페이지 내부 워크플로우

```
1. [F004] Encounter 생성
    ↓
2. [F031] 환자 대면 모드 활성화
    ↓
3. [F034] Problem Unit 생성 (신규) 또는 선택 (재진)
    ↓ (재진시)
4. [F035] 이전 진료와 변화 비교 표시
    ↓
5. S-T-D 입력 단계 (클릭 기반 선택 우선)
    ├── [F005] Situation 입력
    │   ├── 기본: 증상(F051)/부위(F052)/onset/심각도 선택
    │   ├── 확장: 신체검사(F019)/ROM(F020)/근력(F021)/특수검사(F022) 연계
    │   └── 연관 제안: 증상-부위 매핑 기반 관련 부위 자동 제안(F055)
    ├── [F006] Treatment 입력
    │   ├── 치료 계획: 표준 프로토콜 기반 선택 + 가설 상태 허용
    │   ├── 처방: 약물/물리치료/주사 다중 방식 조합
    │   └── S축 연동: 입력된 증상-부위에 따른 치료 옵션 필터링
    └── [F007] Duration 입력
        ├── 회복 예측: 부위별/중증도별 표준 기간 선택
        ├── 재진 일정: 치료 방식 기반 자동 계산

    ↓
```

### 자유 텍스트 정책 결정의 쟁점

**클릭 기반 선택 우선 원칙**과 **완전한 구조화의 한계** 사이에서 다음과 같은 정책 결정이 필요합니다:

#### S축 구조화의 현실적 한계

1. **증상 마스터(F051) 한계**: 정형외과 50개 증상으로 커버되지 않는 복합 증상 표현
2. **부위 마스터(F052) 한계**: 50개 세분화로도 표현되지 않는 미세 해부학적 위치
3. **검사 결과 다양성**: ROM/근력 측정의 개별적 특수 상황 및 예외적 소견

#### T축 구조화의 임상적 한계

4. **치료 프로토콜 한계**: 표준 치료로 분류되지 않는 개별 맞춤 치료
5. **가설 치료의 불확실성**: 진단 확정 전 시도적 치료의 기록 방식
6. **다학제 치료 조합**: 여러 치료 방식의 복합 적용 시 선택 체계의 한계

#### D축 구조화의 예측적 한계

7. **개별 회복 패턴**: 환자별 특수 상황으로 인한 표준 기간과의 차이
8. **외부 요인 변수**: 환자 생활 패턴, 직업적 요인 등이 경과에 미치는 영향
9. **복합 문제의 시간축**: 여러 Problem이 서로 영향을 미칠 때의 기간 예측

#### 구조화 vs 사용성 균형점

10. **보완적 자유 텍스트**: 선택 항목 + 추가 기술 조합의 허용 범위
11. **예외적 자유 텍스트**: 구조화 실패 시 전면 자유 기술로 대체하는 조건
12. **구조화 유도 수준**: 자유 텍스트 입력 시 구조화 요소 추출 시도 여부
13. **데이터 품질 vs 진료 연속성**: 완전 구조화 요구와 진료 흐름 방해 최소화 간 우선순위
