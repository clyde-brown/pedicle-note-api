# 페디클 STD 권장 구성안 (PRD/문제정의서용)

## 1. Situation (S) – 환자가 느끼는 현재 상태

### 1.1 필수 항목 (최소 진료 성립 조건)

* 해부학적 부위
* 좌/우/양측 구분
* 주 증상의 강도(정도)

> 목적:
>
> * “어디가, 얼마나” 아픈지는 **모든 진료의 공통분모**

---

### 1.2 선택 항목 (진료 맥락 강화)

* 증상 유형(통증 양상)
* 증상 발생 시점 또는 기간
* 악화 요인
* 완화 요인

> 목적:
>
> * 질환 추론 정확도 향상
> * ICD 자동 제안의 입력 신호 강화

---

### 1.3 확장 항목 (필요 시만)

* 유발 동작 또는 계기
* 하루 중 변화 패턴
* 이전 치료 경험에 대한 환자 진술

> 정책 메모:
>
> * 기본 화면에서는 비노출 가능
> * 재진, 만성 통증, 설명 필요 케이스에서만 활용

---

### 1.4 자유 서술 (정책 미정)

* 환자가 표현한 주관적 불편 설명 (한 줄)

> 주의:
>
> * 구조화 실패를 보완하는 **보조 수단**
> * STD의 중심은 아님

---

## 2. Treatment (T) – 의사의 판단과 개입

### 2.1 필수 항목

* 치료 또는 처치의 종류
* 치료 카테고리(약물, 물리치료, 주사, 교육)

---

### 2.2 선택 항목

* 처방 약물 요약
* 치료 강도 또는 횟수
* 치료에 대한 가설 여부(확정/비확정)

> 목적:
>
> * “지금은 가설 단계임”을 명시적으로 남길 수 있어야 함

---

## 3. Duration (D) – 시간과 경과 관리

### 3.1 필수 항목1

* 재진 필요 여부

---

### 3.2 필수 항목2

* 다음 내원 예정 시점

> 목적:
>
> * 진료를 “오늘”로 끝내지 않고 **시간 축으로 연결**

---

## 4. STD 공통 규칙 (문서에 반드시 명시)

아래 문장은 **그대로 문서에 넣는 걸 추천**합니다.

* STD는 **의미 단위 기준**으로 정의한다
* 입력 방식(UI, 클릭/텍스트 여부)은 본 문서에서 정의하지 않는다
* 필수 항목만으로도 진료 저장은 가능해야 한다
* 선택/확장 항목은 진료 흐름을 방해하지 않는 방식으로 제공한다
