우리 프로젝트 landing-builder-api의 전체 MVP 범위를 정의하는 마스터 PRD를 작성해줘.

[작업 목표]
테넌트가 페이지를 생성하고, 다양한 섹션을 조립하여 퍼블리싱하며,
Next.js가 이를 소비하는 전체 생애주기를 단일 PRD 문서로 정의할 것.

[포함될 주요 기능 범위]

- 테넌트 & 페이지 관리:
  테넌트별 페이지 생성, 목록 조회, 페이지 설정 변경.

- 섹션 편집 엔진:
  페이지 내 섹션 추가/삭제/순서 변경 및
  섹션 타입별 상세 데이터(Hero, Feature, CTA 등) 편집.

- 라이프사이클 관리:
  Draft 상태 저장 및 Published 스냅샷 생성(퍼블리시),
  Draft / Published 명확한 분리.

- 데이터 서빙:
  Next.js SSG 빌드를 위한
  Published Snapshot 기반 최적화 데이터 제공.

[핵심 요구사항]

- docs/convention/business-rules.md에 정의된
  **모든 비즈니스 컨벤션(CONV-A ~ CONV-G)**을
  전제 조건으로 깔고 PRD를 작성할 것.
  (특히 Table-per-Section 원칙과 Draft / Publish 분리를 강하게 준수)

- PRD는 prd-writer의 출력 템플릿
  (프로젝트 핵심 → 사용자 여정 → 기능 명세 → 메뉴 구조 →
   페이지별 상세 기능 → 데이터 모델 → 기술 스택)
  순서를 반드시 따를 것.

- 모든 기능은 F001부터 시작하는 기능 ID(Fxxx)를 부여하고,
  [기능 명세] ↔ [메뉴 구조] ↔ [페이지별 상세 기능] 간
  정합성을 완벽하게 유지할 것.

- 컨벤션(CONV-*) 위반 사항이 발견되면,
  해당 설계를 수정하여 위반이 없는 상태로 PRD를 완성할 것.

[출력]

- 결과물은 planning/prd.md 파일에 저장할 것.
- 파일이 이미 존재할 경우, 전체 내용을 덮어쓸 것.
